# 更新日志

## Unreleased

- 冻结 ReadyType 1.4.0 热门词包的数据源与发布边界：使用 Wikimedia Pageviews 和 Wikidata，每日离线生成并通过签名词包发布，不接入未取得商业许可的 TMDB 等数据源。
- 新增中英文词包生成与 AI 整理方案，明确 AI 只负责维护侧分类复核和歧义标记，不能进入实时输入链路、使用用户 API Key 或直接发布新词条。
- 将热门词运行时候选收紧为每次 10-20 个，并把个人纠正记忆与确认式学习明确顺延为 1.5.0 候选。
- 重画 1.4.0 热门词包原型，将功能并入现有「语音识别」页面，移除独立导航和面向用户的分类管理负担。
- 新增热门词包协议、SHA-256 与 Ed25519 校验、尺寸/过期/最低版本约束，以及“不可变版本目录 + 原子活动指针 + 上一有效版本回退”的本地存储核心；当前阶段不接网络、不改变现有识别链路。
- 热门词现可作为低优先级来源并入统一词典：用户常用词与内置词优先，单次最多选择 20 个热门词，单词过期不影响整包，公开别名只参与识别提示、不直接自动改写用户原话；5000 词压力测试继续满足候选延迟门槛。
- 新增热门词包后台更新核心：仅允许 HTTPS 同源 manifest 与词包文件，每日最多自动检查一次，支持 ETag 和手动强制检查；下载、签名或写入失败时继续使用旧包，本地包缺失但服务器返回 304 时会自动无条件重取。
- 新增生产 Ed25519 公钥、Wikimedia/Wikidata 确定性候选生成器、独立签名器和 GitHub Pages 发布工作流；发布前必须使用 App 自身校验器通过真实产物验签，生产私钥仅保存在维护者钥匙串和 GitHub Secret。
- 热门词现已接入 App 的共享内存词典：启动时读取本地有效包，空闲 8 秒后及后续每 24 小时后台检查，录音路径不访问网络；设置页可查看用户化状态并手动更新，失败时继续使用上一有效包。
- 当前 1.4.0 开发构建升为 `1.4.0 (88)`，用于真实加载最低兼容版本为 1.4.0 的线上词包。
- 完成 1.4.0 热门词真实语音验收：近期名称和普通表达误伤控制通过；记录长句中英技术词的一次偶发退化及短句复测通过结果，不针对单词追加临时替换补丁。

## 1.3.0 - 2026-07-16

- 新增 ReadyType 1.3.0 中英文需求、匿名事件规范和实施计划，明确公开源码构建默认不发送统计，且禁止上传语音、文字内容、窗口标题、常用词、剪贴板和 API Key。
- 新增强类型匿名事件层、用户开关和 No-op 默认实现，覆盖启动、输入开始、完成、取消及固定错误码；未注入 App ID 的源码构建仍不会发送网络统计。
- 优化常用词候选排序的上下文与排序键预计算，使 2,000 词压力测试的 P95 稳定低于 50ms 门槛。
- 接入 TelemetryDeck 官方 Swift SDK 作为可选统计 Provider；App ID 仅支持构建时注入，缺失配置时继续使用 No-op，并关闭自动 Session 事件和 Session 统计。
- 构建脚本会复制并校验 TelemetryDeck 隐私清单资源包，避免自定义 `.app` 打包遗漏第三方隐私声明。
- 新增不绑定版本号的本地发布门禁和八页面 UI 冒烟检查，统一验证正式统计配置、测试、签名结构、ZIP、DMG、隐私与敏感信息。
- 收轻紧凑语音胶囊的标题与字重，并让波形直接响应麦克风实际音量：安静时保持低位，说话时随音量变化，且不启动第二路录音。
- 识别与整理阶段改为 Typeless 式紧凑白色胶囊，移除无关的计时、模式徽标、彩色流光和波形；录音阶段继续使用真实音量波形。
- HUD 全流程统一为固定白色胶囊；新增按识别与整理阶段推进的 Thinking 进度条，并保持录音波形、状态和结果切换时外壳不跳变。
- 新增 ReadyType 原创双音上扬激活提示音；只有权限通过并即将开始录音时播放。
- `scripts/build-app.sh` 现在会在构建前清除 `dist` 中旧 ZIP、旧 DMG、重复 App 和 Finder 元数据，只保留当前应用。
- HUD 在 macOS 26 使用系统原生液态玻璃材质，并为旧系统提供自适应材质降级；录音态右上角新增与 `Esc` 共用取消路径的关闭按钮，每天第一次使用显示 1.6 秒“按 Esc 退出”提示，悬停按钮也可查看。
- 视觉验收使用独立偏好域，不再占用用户当天第一次 `Esc` 提示记录。
- HUD 在保留现有尺寸与交互的基础上改为 Typeless 式黑白液态玻璃配色，移除容易显脏的灰白覆盖层；文字、进度和操作图标统一为白色，波形仅保留中间绿色作为 ReadyType 状态点缀。
- 正式发布构建升为 `1.3.0 (87)`。

## 1.2.0 - 2026-07-13

- 常用词现在同时参与极速识别和高精度 Whisper 识别，并以受限、去重的标准写法提示 AI 整理，避免保存后只在部分链路生效。
- 重复添加同一个词时可更新其规范大小写，且不会覆盖既有分类、别名或其他学习信息。
- 修复常用词候选任务超时后仍继续计算的问题，避免较慢环境下取消不及时和 CI 性能门禁误报。
- 新增按版本 tag 自动创建 GitHub Release 的流程，发布前会校验版本、运行测试、扫描敏感信息，并上传 ZIP、DMG 和 SHA-256 校验文件。
- 中文及中英混合输出统一使用全角中文标点，全英文输出统一使用半角英文标点；版本号、网址、时间和数字格式保持不变。
- “整理成文”进一步要求恢复句内及句末标点，避免口语输出连成一段。

### 新增

- 包含空格的常用词支持经用户确认后拆分为多个独立词条，便于修正误把多个词输入在一行的情况。
- 新增本机使用概览，展示累计语音时长、完成输入次数、输出字数、预计节省时间和最近 14 天趋势；只保存每日聚合统计，不保存转写正文。
- 新增“中文文字”设置，支持简体中文、繁体中文和跟随系统；默认使用简体中文，并统一应用于直接转文字和 AI 输出。
- 新增 ReadyType 1.2.0 UI/UX Refresh 中英文需求、交互架构、视觉动效规范和实施计划。
- 新增跟随系统、浅色和深色三种可持久化外观，主窗口、HUD 与菜单栏浮窗共享外观设置。

### 变更

- 常用词输入区明确说明空格属于词语本身；多个词需使用换行、逗号、顿号或分号分隔，避免误解并保护多词产品名。
- 语音输入胶囊缩小为更紧凑的固定尺寸，精简状态灯、计时器和波形，并移除重复的场景徽标，保留必要状态与输出方式。
- 调整侧栏顺序：首页置于首位，使用概览紧随其后，使默认打开页面与导航层级一致。
- 修复菜单栏浮窗点击外部后不易收回的问题，并支持按 Esc 关闭。
- 使用概览支持清除本机统计；连续使用天数在当天尚未输入时保留截至昨天的有效连续记录。
- 统一菜单栏浮窗的 AppKit 容器与 SwiftUI 内容尺寸，避免打开时重复布局造成卡顿；外观设置应用到整个浮窗。
- 首页在高精度识别已启用并准备好时使用绿色状态点，同时保持录音、识别和错误状态的优先级。
- 当前开发构建升为 `1.2.0 (70)`，结束后续测试包继续复用旧构建号的问题。
- 高精度识别启用 VAD 分段，并拦截由相同长片段连续重复组成的异常识别结果，避免 Whisper 幻觉文本被直接粘贴。
- 将热门词包及其 UI 原型从 1.2.0 顺延到 1.4.0，避免与 UI/交互重构同时修改识别候选架构。
- 主窗口调整为首页、常用词、语言与输出、快捷键、语音识别、权限与隐私、关于七个入口，设置内容按功能拆分。
- 首页移除模式和场景控制面板，改为运行状态、快捷键、高精度语音包、默认输出和最近结果摘要。
- HUD 改为固定尺寸的自适应玻璃胶囊，流光限制在顶部局部边缘，并保留减少动态效果降级。
- 优化首页与“关于 ReadyType”的产品说明，移除侧栏英文占位文案，并提高浅色侧栏与选中态的对比度。
- 高精度语音包更新状态不再向用户显示模型版本日期，但内部仍保留版本比较和安全更新逻辑。
- 高精度语音包无更新时明确显示“已安装最新语音包，无需更新”，避免“当前版本”的指代歧义。
- 重写常用词页面说明，用具体误识别示例解释作用、提醒时机和保存边界，并将批量导入改为用户更易懂的“一次添加多个”。
- 常用词批量添加支持换行、中文/英文逗号、顿号和分号，逐词去重并独立保存；自动迁移此前误存的逗号组合词。
- 常用词手动添加不要求配置适用场景，ReadyType 根据当前 App、分类和上下文自动调整使用优先级。
- “语言与输出”按默认输出、AI 功能和输入到当前 App 三个用户任务重新组织，DeepSeek 服务地址和模型名称收进高级设置。
- 修复“跟随系统”可能只更新标题栏的问题，统一 AppKit 与 SwiftUI 的有效外观；浅色改为分层灰白配色，并为主题切换增加柔和过渡；侧栏图标使用固定列宽，确保全部标题左对齐。

## 1.1.0 - 2026-07-11

### 新增

- 新增 ReadyType 1.4.0 中英文需求和计划文档，定义“热门词包”的分层词库、后台更新、本地缓存、过期和性能边界。
- 新增 ReadyType 1.4.0 热门词包中英文交互图，覆盖用户可见设置、后台更新、语音输入候选词决策和设置页信息结构。
- 新增 ReadyType 1.4.0 热门词包 HTML UI 原型，用于后续 SwiftUI 设置页实现参考。
- 新增高精度语音包真实在线版本检查：从 ReadyType 公共仓库读取受控清单，可区分尚未检查、检查中、未安装、当前推荐版本、发现新版和暂时无法检查。
- 新增高精度语音包安全更新流程：按清单指定的 WhisperKit 官方模型安装，成功后持久化版本并清理旧包，失败时保留当前可用语音包。
- 新增 1.1.0 本地发布门禁记录，覆盖单元测试、构建、zip、DMG、UI 文案、TextEdit 粘贴、常用词 UI、视觉截图和敏感信息检查。
- 常用词分类新增“公司/组织”，并将默认分类文案从“通用”调整为更面向用户的“其他”。
- 新增 ReadyType 1.1.0 中英文需求说明和版本索引，区分已有基础能力、当前缺口、验收标准和非目标。
- 新增 ReadyType 1.1.0 中英文版本计划，明确常用词库、确认式学习建议、App 场景语气、快捷键自定义和高精度语音包更新提示的范围。
- 新增可直接转发给试用用户的中文测试邀请文案。
- 新增公开 Roadmap 和测试说明，方便试用用户理解当前版本边界、反馈方式和后续方向。
- 新增故障排查文档，覆盖未签名首次打开、权限、快捷键、粘贴降级、DeepSeek 连接检查、高精度语音包准备状态和反馈方式。

### 变更

- 常用词 UI 验收改用独立临时词库并移除依赖时序的通知注入，不再接触用户真实词库数据。
- 非邮件的整理成文和翻译成英文不再无根据添加问候、致谢、确认、署名或收尾；邮件场景仍保留用户需要的礼貌落款能力。
- 将 1.4.0 热门词包 UI 原型调整为更接近 Typeless 的轻玻璃风格，并加入浅色/深色切换按钮。
- 过滤内部 UI 验收使用的常用词测试数据，避免诊断词条残留在真实用户常用词列表中。
- 将高精度语音包状态调整为“当前推荐版本”，只在远程清单与本机版本一致时显示。
- 高精度语音包清单增加版本信息，设置页新增独立“更新状态”和“检查更新”入口，避免和“是否准备好”混在一起。
- 加强“翻译成英文 + 邮件”输出约束：保留明确收件人、按要求输出编号列表，且未要求主题时不主动添加主题行。
- 确认式常用词建议会过滤过长候选和口头结束词，避免把完整句子、私人正文或“OK / 好了 / 完成”等口头噪声加入建议。
- 将常用词建议相关文案从“记忆建议 / 记住写法”调整为“常用词建议 / 加入常用词”，避免用户误解为静默学习。
- 在产品描述中突出 AI 输出默认调用 DeepSeek V4 Flash，并说明按当前官方 API 价格和日常用量估算成本很低。

## 1.0.0 - 2026-06-24

ReadyType 首个公开候选版本。

### 新增

- GitHub Issue 模板和 README 反馈入口，覆盖安装、权限、快捷键、粘贴、识别准确率和输出语气问题。
- 中文优先的 macOS 语音输入，双击 `Option` 开始和完成。
- 语音输入中按 `Esc` 取消。
- 支持直接转文字、整理成文、翻译成英文和写给 AI 四种输出方式。
- 使用 DeepSeek 处理需要 AI 的输出，密钥存储在 macOS 钥匙串。
- 自动识别路由：优先保持快速响应，适合时使用更准确的本机识别。
- 高精度语音包下载、准备、状态展示和删除入口。
- 常用词和确认式个性化建议，用于改善术语较多的输入。
- 菜单栏浮窗、主控制台、设置、权限、新用户引导和低打扰语音输入浮层。
- 自动粘贴，失败时复制到剪贴板。
- `.app`、`.zip` 和 `.dmg` 打包脚本。

### 说明

- 1.0.0 分发包未签名、未公证。
- 语音识别不需要额外填写云端语音 API 密钥。
- AI 输出方式会把当前文本发送给 DeepSeek 处理。
