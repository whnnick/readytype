# Roadmap

ReadyType 1.0.0 已经可以作为公开测试版本使用。接下来的目标是把语音输入体验从“能用”继续打磨到更稳定、更懂场景、更少打扰。

## 当前版本

- 中文优先的 macOS 语音输入。
- 双击 `Option` 开始和完成，`Esc` 取消。
- 支持直接转文字、整理成文、翻译成英文和写给 AI。
- DeepSeek V4 Flash 作为默认 AI 输出模型。
- 高精度语音包、自动识别路由和常用词建议。
- 自动粘贴，失败时复制到剪贴板。
- 未签名、未公证的 DMG 分发包。

## 下一阶段重点

1.1.0 的详细计划见 [ReadyType 1.1.0 计划](./versions/1.1.0/PLAN.zh-CN.md)。

### 常用词库

让用户维护自己的常用词，例如人名、项目名、产品名、技术词和公司名。目标是减少专业词、大小写和中英混说时的识别错误。

### 自学习建议

在用户确认后，把反复修正过的词加入常用词库。该功能需要明确可控，不能静默收集私人内容，也不能让错误修正污染词库。

### 场景语气

根据当前 App 和用户选择的写作场景调整输出语气。聊天场景应更自然，邮件和文档场景应更清楚完整，AI 工具场景应更像任务说明。

### 快捷键自定义

允许用户把默认的双击 `Option` 改成其他触发方式，减少与个人输入法或系统习惯冲突。

### 高精度语音包更新

提供更清楚的状态、版本、更新提示和重新下载入口。目标是让用户知道当前是否已经准备好，以及什么时候需要更新。

### 语音输入浮层

继续优化 HUD 的视觉反馈和动效，探索更接近 macOS 液态玻璃和低打扰系统浮层的表现。

## 不优先做

- 多个大模型供应商配置。
- 面向工程师的复杂模型市场。
- 云端语音识别 API 配置。
- 大量高级设置项。

ReadyType 会继续保持中文优先、配置尽量少、日常输入成本低的方向。
