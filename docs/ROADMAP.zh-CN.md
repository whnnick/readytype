# Roadmap

ReadyType 1.3.0 已经建立隐私优先的匿名产品统计。接下来的目标是用真实使用数据继续提升输入准确率、响应速度和场景适配。

## 当前版本

- 中文优先的 macOS 语音输入。
- 双击 `Option` 开始和完成，`Esc` 取消。
- 支持直接转文字、整理成文、翻译成英文和写给 AI。
- DeepSeek V4 Flash 作为默认 AI 输出模型。
- 高精度语音包、自动识别路由和常用词建议。
- 自动粘贴，失败时复制到剪贴板。
- 未签名、未公证的 DMG 分发包。

## 版本规划

1.2.0 见 [ReadyType 1.2.0](./versions/1.2.0/README.zh-CN.md)：重构 UI 与交互，让 HUD 成为主要体验，主窗口退居管理，并加入跟随系统、浅色和深色外观。

1.3.0 见 [ReadyType 1.3.0](./versions/1.3.0/README.zh-CN.md)：建立不收集用户内容的匿名产品统计，用于验证首次使用、留存、功能采用率、失败率和等待时间。

1.4.0 见 [ReadyType 1.4.0](./versions/1.4.0/README.zh-CN.md)：使用 Wikimedia Pageviews 与 Wikidata 生成并签名“热门词包”，让近期电影、科技产品、赛事等公开热门词在后台更新并低优先级参与识别；AI 只在发布前离线辅助整理。

### UI 与交互重构

默认自动理解当前 App 和语义，减少“输出方式”“写作场景”“识别方式”等同时出现在主界面的选择。语音浮层提供稳定、低打扰且可解释的状态反馈。

### 常用词库

让用户维护自己的常用词，例如人名、项目名、产品名、技术词和公司名。目标是减少专业词、大小写和中英混说时的识别错误。

### 自学习建议

在用户确认后，把反复修正过的词加入常用词库。该功能需要明确可控，不能静默收集私人内容，也不能让错误修正污染词库。

个人纠正记忆与跨使用次数学习作为 1.5.0 候选，不进入 1.4.0 热门词包的发布范围。

### 场景语气

根据当前 App 和用户选择的写作场景调整输出语气。聊天场景应更自然，邮件和文档场景应更清楚完整，AI 工具场景应更像任务说明。

### 快捷键自定义

允许用户把默认的双击 `Option` 改成其他触发方式，减少与个人输入法或系统习惯冲突。

### 高精度语音包更新

提供更清楚的状态、版本、更新提示和重新下载入口。目标是让用户知道当前是否已经准备好，以及什么时候需要更新。

### 热门词包

后台更新由 Wikimedia Pageviews 和 Wikidata 生成的近期常用词，用于提升电影名、产品名、赛事名等专有名词识别。该功能必须签名校验、本地缓存、可关闭、可删除、可过期，并且不上传用户输入内容。

### 语音输入浮层

在 1.2.0 中完成 HUD 全状态重构，采用原生材质、局部边缘流光和稳定布局，并支持浅色、深色和减少动态效果。

## 不优先做

- 多个大模型供应商配置。
- 面向工程师的复杂模型市场。
- 云端语音识别 API 配置。
- 大量高级设置项。

ReadyType 会继续保持中文优先、配置尽量少、日常输入成本低的方向。
