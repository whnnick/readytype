# ReadyType 1.2.0 视觉与动效规范

## 视觉定位

系统级、克制、清晰、轻科技感。参考 Typeless 的低打扰体验和 Apple 的材质层次，但保留 ReadyType 自己的品牌颜色、图标和中文表达。

## 主题

- 跟随系统：默认。
- 浅色：浅灰画布、白色内容层、深色文字、低饱和绿色状态色。
- 深色：接近系统黑的画布、抬升的深灰内容层、高对比文字。
- 同一语义 token 在两套主题中保持一致，不在视图内散落硬编码颜色。

## HUD

- 高度目标：56-64 pt；宽度根据内容设置稳定上下限。
- 圆角：接近半高，但保留轻微直边感，不做夸张药丸。
- 材质：优先 AppKit/SwiftUI 原生玻璃材质；降级系统保持纯色半透明层。
- 边缘：一层基础描边、一段局部高光、一段状态流光。
- 流光：沿长边缓慢位移，不绕边框转圈，不覆盖内容。
- 阴影：只用于从当前 App 内容中分离，避免彩色大光晕。

## 动效状态

| 状态 | 动效 |
| --- | --- |
| 出现 | 10-12 pt 上浮，160-200 ms 淡入 |
| 正在听 | 波形响应音量，边缘高光缓慢流动 |
| 正在识别 | 波形收敛，文本 120-160 ms 交叉淡入 |
| 正在整理 | 边缘色温轻微变化，不改变布局 |
| 已完成 | 单次绿色确认，约 900 ms 后淡出 |
| 已复制 | 琥珀色提示，约 1.5 s 后淡出 |
| 出错 | 单次轻微水平反馈，随后静止 |

## 排版和控件

- 使用系统字体，不缩放字号跟随窗口宽度。
- 主窗口标题、分组标题和辅助说明建立三级层次。
- 二元设置使用开关，多选使用菜单，外观使用三段式选择。
- 图标优先 SF Symbols；不使用文字胶囊替代常见图标按钮。
- 卡片圆角不超过 8 pt，避免卡片套卡片。

## 无障碍

- 文本与背景达到可读对比度。
- 颜色不作为唯一状态信号。
- 支持键盘导航、VoiceOver 标签和减少动态效果。
- HUD 文案在最窄尺寸下不截断关键动作。
