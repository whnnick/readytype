# ReadyType 1.2.0 需求说明：UI/UX Refresh

## 背景

1.1.0 已完成核心语音输入能力，但主窗口同时展示输出方式、写作场景、识别方式和语音包状态，用户需要理解过多内部概念。1.2.0 应减少决策和技术文案，让用户形成“触发、说话、得到结果”的稳定心智模型。

## 产品目标

1. 双击快捷键后 100 ms 内出现可感知反馈。
2. 默认无需先选择输出方式或写作场景。
3. 用户在聊天、邮件、文档和 AI 工具中获得适合当前 App 的输出。
4. 任一状态都能看懂：正在听、正在识别、正在整理、正在输出、已完成、已取消、出错。
5. 浅色和深色下均保持清晰对比，并支持跟随系统。
6. 开启“减少动态效果”后，位移、扫光和抖动降级为淡入淡出。

## 信息架构

主窗口左侧导航调整为：

1. 首页
2. 常用词
3. 语言与输出
4. 快捷键
5. 语音识别
6. 权限与隐私
7. 关于

首页只展示可用状态、快捷键、高精度识别状态、最近结果和需要处理的事项，不继续承担完整设置面板职责。

## 常用词模块

### 用户目标

用户把容易识别错的人名、品牌、产品、项目和专业词提前告诉 ReadyType，后续语音输入更容易得到正确写法。常用词不是完整历史，也不会静默学习用户修改。

### 数据单位

- 一个正确写法对应一条独立记录，例如 `ChatGPT`、`Codex`、`GitHub Actions`。
- 词语内部空格属于内容，不作为分隔符。
- 每条记录包含正确写法、分类、适用场景，以及可选的常见误识别写法。
- 去重忽略大小写和首尾空白，但不合并不同的完整词。

### 添加方式

- “添加一个”：一次只能填写一个完整词；检测到分隔符时不保存，并引导用户使用批量添加。
- “一次添加多个”：支持换行、中文/英文逗号、顿号、中文/英文分号；保存前拆成独立词并去重。
- 已有的逗号组合错误数据在加载时自动拆分、去重并持久化，之后每个词可单独删除。

### 生效逻辑

- 用户保存的词优先于内置词，并按当前 App 和写作场景选取相关候选。
- 正确写法直接参与识别上下文；常见误识别只用于受约束的纠错，不做无条件替换。
- “发现可能写错的专有词时提醒我”只控制完成输入后的建议；建议必须由用户确认后才保存。
- 删除后立即从后续识别候选中移除。

## 自动化与用户控制

- 默认输出行为：自动识别 App 和语义并整理成适合发送的文本。
- “直接转文字”“翻译成英文”“写给 AI”保留为明确的次级操作。
- 自动判断必须可解释：HUD 可短暂显示“微信 · 自然聊天”“邮件 · 自动排版”等标签。
- 判断不确定时使用通用整理，不猜测收件人、事实、语气或格式。
- 高级用户仍可在“语言与输出”中设置默认行为。

## HUD 状态

- 准备：轻微浮现，不播放循环流光。
- 正在听：波形响应音量，边缘流光低速流动。
- 正在识别：波形收敛，保持稳定尺寸。
- 正在整理：标题交叉淡入，不改变胶囊宽高。
- 正在输出：展示目标 App，不暴露内部粘贴实现。
- 已完成：绿色边缘确认，约 900 ms 后退出。
- 已复制：提示已复制，保留约 1.5 秒。
- 已取消：快速淡出，不出现错误色。
- 出错：单次轻微反馈和可读恢复建议，不显示技术异常栈。

## 外观

- 默认：跟随系统。
- 可选：浅色、深色。
- HUD 使用原生材质、边缘高光和有限色彩，不使用大面积渐变背景或持续旋转边框。
- 主窗口保持原生 macOS 密度，不设计成营销落地页。

## 非目标

- 不像素级复制 Typeless。
- 不修改 ASR 路由、DeepSeek 协议、粘贴策略和权限模型。
- 不加入完整转写历史、账号系统或云同步。
- 不在本版本实现 1.3.0 热门词包。

## 验收标准

- 新用户无需理解“场景”和“模式”即可完成首次输入。
- 微信、邮件、文档、AI 工具至少四类 App 有明确但低打扰的场景反馈。
- HUD 全状态尺寸稳定，无文字截断和跳动。
- 浅色、深色、跟随系统均通过截图和真实 App 验收。
- `Esc`、自动粘贴、复制降级、高精度识别和 DeepSeek 行为无回归。
