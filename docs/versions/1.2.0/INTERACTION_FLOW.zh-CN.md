# ReadyType 1.2.0 交互图：热门词包

## 1. 用户可见交互

目标：默认无感启用；用户只有在设置里查看或管理时才看到热门词包。

```mermaid
flowchart TD
    A["用户打开 ReadyType"] --> B["控制台保持现有输入体验"]
    B --> C{"用户是否进入设置"}
    C -- "否" --> D["后台按空闲状态检查热门词包"]
    D --> E["用户继续双击 Option 语音输入"]
    C -- "是" --> F["设置 > 热门词包"]
    F --> G["显示状态：已更新 / 正在更新 / 暂时无法更新 / 已关闭"]
    G --> H{"用户操作"}
    H -- "保持默认" --> I["自动更新开启，后台处理"]
    H -- "立即更新" --> J["开始后台更新，不阻塞输入"]
    H -- "关闭自动更新" --> K["停止后续自动检查，保留已有本地词包"]
    H -- "删除词包" --> L["删除本地热门词包，候选立即移除"]
    J --> G
    K --> G
    L --> G
```

## 2. 后台更新交互

目标：更新失败不弹窗、不打断录音、不影响自动粘贴。

```mermaid
stateDiagram-v2
    [*] --> WaitingIdle: App 启动后延迟检查
    WaitingIdle --> Skipped: 正在录音/识别/整理/输出
    Skipped --> WaitingIdle: 回到空闲
    WaitingIdle --> CheckNeeded: 距离上次检查超过 1 天
    WaitingIdle --> NoCheck: 今日已检查
    CheckNeeded --> Downloading: 自动更新开启
    CheckNeeded --> Off: 自动更新关闭
    Downloading --> Updated: 下载成功且 hash 有效
    Downloading --> KeepOld: 下载失败但已有旧包
    Downloading --> Unavailable: 下载失败且无本地词包
    Updated --> [*]: 设置页显示已更新
    KeepOld --> [*]: 继续使用旧包，设置页显示暂时无法更新
    Unavailable --> [*]: 不参与候选，设置页显示暂时无法更新
    NoCheck --> [*]
    Off --> [*]
```

## 3. 语音输入时的候选词决策

目标：热门词只做低优先级补充，不能污染用户词和普通表达。

```mermaid
flowchart TD
    A["用户双击 Option 开始输入"] --> B["读取当前 App 和写作场景"]
    B --> C["加载内存中的词库索引"]
    C --> D["合并候选：用户词 + 确认建议 + 内置词 + 场景词 + 热门词"]
    D --> E["过滤过期热门词"]
    E --> F["按优先级排序"]
    F --> G["用户词最高优先级"]
    F --> H["热门词低优先级，只在相关场景加分"]
    G --> I["裁剪 Top N，总数不超过 100"]
    H --> I
    I --> J{"识别路线"}
    J -- "Apple Speech" --> K["传入 contextualStrings"]
    J -- "高精度语音包" --> L["不承诺实时热词偏置，仅保守后处理"]
    J -- "AI 输出模式" --> M["把高相关词作为术语提示传给 DeepSeek"]
    K --> N["生成转写和最终输出"]
    L --> N
    M --> N
    N --> O["自动粘贴或复制降级"]
```

## 4. 设置页信息结构

```mermaid
flowchart LR
    A["设置"] --> B["识别方式"]
    A --> C["DeepSeek"]
    A --> D["输出方式"]
    A --> E["常用词"]
    A --> F["热门词包"]
    F --> G["自动更新热门词开关"]
    F --> H["状态和最后更新时间"]
    F --> I["立即更新"]
    F --> J["删除词包"]
    F --> K["说明：不会上传你的输入内容"]
```

## 交互原则

- 正常语音输入路径不增加任何弹窗。
- 热门词包更新不能阻塞双击 `Option` 后的输入。
- 设置页只展示用户能理解的状态，不暴露 API、manifest、hash 等技术细节。
- 删除词包只影响热门词候选，不影响用户常用词。
- 关闭自动更新后，已下载词包可以继续使用；如果用户删除，则不再参与候选。
