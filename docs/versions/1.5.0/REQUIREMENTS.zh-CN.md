# ReadyType 1.5.0 需求说明

## 问题

现有侧边栏同时放置高频功能和低频配置，入口过多，首页层级不够清楚。用户需要快速进入输入状态、查看使用情况或维护常用词，而不应长期看到连接、快捷键、权限等配置入口。极速识别还需要直接启用系统听写的自动标点，避免自然中文被输出为缺少句内停顿的一整段文字。

## 需求

1. 主导航最多保留首页、使用概览、常用词和设置四个入口。
2. 设置入口固定在侧边栏底部，并使用通用齿轮图标。
3. 设置页提供通用、语音识别、快捷键、权限与隐私、关于五个稳定分类。
4. 外观选择从侧边栏移入通用设置。
5. 高精度语音包引导仍能直接打开设置中的语音识别分类。
6. 深色、浅色及跟随系统外观都必须保持可用。
7. 本次只调整导航和页面归属，不删除功能，不改变用户数据。
8. 极速识别必须启用 macOS 系统自动标点；中文及中英混合输出继续使用全角标点，全英文输出继续使用半角标点。

## 验收标准

- 左侧不再显示语言与输出、快捷键、语音识别、权限与隐私和关于等独立入口。
- 所有迁移后的功能仍能在设置中访问。
- 导航结构有单元测试和真实应用 UI 冒烟验证。
- 系统听写请求的自动标点配置有单元测试，且不得用固定短语替换规则推断句内标点。
