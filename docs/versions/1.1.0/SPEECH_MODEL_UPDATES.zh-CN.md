# 高精度语音包更新设计

## 产品目标

让用户能准确知道高精度语音包是否可用、是否已经准备完成，以及是否有经过 ReadyType 验证的新版本。检查和更新都不能打断当前语音输入。

## 用户可见状态

三类状态独立管理，不能互相覆盖：

- 安装状态：未安装、下载中、已安装、安装失败。
- 使用状态：尚未准备、准备中、已准备好。
- 更新状态：尚未检查、检查中、当前推荐版本、发现新版、暂时无法检查。

“已准备好”只表示当前已安装语音包已加载完成，不代表它是最新版本。“当前推荐版本”只表示远程清单与本机版本一致。

## 更新流程

1. 用户点击“检查更新”，App 从 ReadyType 公共仓库读取一个小型 JSON 清单。
2. 清单只描述 ReadyType 已验证的 WhisperKit 官方模型 variant、目录名、版本和大小，不允许提供任意下载地址。
3. 如果发现新版，用户明确点击更新后才下载约 626 MiB 的语音包。
4. 新语音包先下载到临时位置并完成可用性检查，再切换为当前版本。
5. 下载或校验失败时保留旧版本，当前语音输入不受影响。
6. 新版本准备完成后再清理旧版本；用户仍可在设置中删除语音包。

## 远程清单

清单由 ReadyType 仓库维护，字段保持最小：

```json
{
  "schemaVersion": 1,
  "recommendedModel": {
    "variant": "large-v3-v20240930_626MB",
    "folderName": "openai_whisper-large-v3-v20240930_626MB",
    "version": "2024-09-30",
    "sizeDescription": "约 626 MiB"
  }
}
```

客户端拒绝未知 schema、空字段、非 HTTPS 来源以及不属于 WhisperKit 官方命名空间的模型。网络失败只显示“暂时无法检查更新”，不会回退成“当前推荐版本”。

## 性能与隐私

- 检查只下载小型 JSON，不加载模型，不阻塞录音和首屏。
- 模型下载继续使用 WhisperKit 官方下载能力，不经过 ReadyType 服务器。
- 检查请求不上传转写文本、常用词、当前 App、窗口标题或 DeepSeek 密钥。
- 第一阶段只支持手动检查；后台定期检查需在稳定验证后另行实现。

## 验收标准

- 远程清单与本机一致时显示“高精度语音包已是当前推荐版本”。
- 远程清单更新时显示新版本与大小，并提供明确更新入口。
- 断网、超时、无效 JSON 和未知 schema 都显示“暂时无法检查更新”。
- 更新失败后旧语音包仍可识别。
- 更新成功后重启 App 仍能识别已安装版本，并可正常预热。
