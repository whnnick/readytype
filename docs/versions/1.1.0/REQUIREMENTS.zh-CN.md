# ReadyType 1.1.0 需求说明

## 背景

ReadyType 1.0.0 已经可以公开测试。下一阶段不是继续堆功能，而是把真实输入中反复出现的问题做成可维护的产品能力：常用词、确认式学习、不同 App 的输出语气、快捷键配置和高精度语音包状态。

当前代码中已经存在一些基础能力。1.1.0 的重点是产品化收口、文案统一、状态一致和验收闭环，不是从零重写。

## 当前已有基础

### 常用词

已有能力：
- `UserVocabularyStore` 支持本地保存、加载、去重、删除和多行导入。
- `SettingsViewModel` 已支持添加、导入、删除常用词。
- `SmartTermDictionary`、`ContextualVocabularyProvider` 和 `TermCorrectionService` 已能把用户常用词用于识别上下文和术语修正。
- 相关测试已覆盖 `UserVocabularyStoreTests`、`SettingsViewModelTests`、`ContextualVocabularyProviderTests`、`TermCorrectionServiceTests`。

当前缺口：
- 设置页的常用词体验需要确认是否足够清楚。
- 分类已补齐“公司/组织”，默认分类文案已调整为“其他”；后续需要继续确认普通用户是否能理解。
- 需要确认普通用户能理解“可能识别成”“所有场景”等说明。

### 确认式学习建议

已有能力：
- 控制台已有“建议加入常用词”列表。
- 用户可以忽略或“记住写法”。
- `UserVocabularySuggestionService` 和 `UserVocabularyLearningService` 已存在。
- 设置中已有“记忆建议”开关。

当前缺口：
- “记忆建议 / 记住写法”已改为“常用词建议 / 加入常用词”；后续需要继续复核控制台和设置页是否足够清楚。
- 需要确认拒绝后冷却、去重、长度限制和口头噪声过滤是否符合产品要求。
- 需要明确不保存完整私人文本。

### App 场景语气

已有能力：
- `OutputScenario`、`OutputContext`、`PromptTemplates` 已承担场景和提示词职责。
- 控制台已有写作场景选择。
- 部分测试覆盖邮件、AI 工具、技术词和整理成文输出。

当前缺口：
- 聊天场景仍可能过于正式。
- 需要明确 App 自动识别和用户手动场景选择的优先级。
- 需要把微信/聊天、邮件、笔记、文档、AI 工具的输出边界写成可测试规则。

### 快捷键自定义

已有能力：
- `VoiceShortcutConfiguration` 已支持双击 `Option`、`Control`、`Command`、`Fn`。
- `SettingsStore` 已保存快捷键配置。
- `GlobalShortcutService` 已按配置触发。
- `Esc` 取消已有测试覆盖。

当前缺口：
- 设置页里的“触发方式”是否足够清楚需要复核。
- 需要确保修改后立即生效，并且失败提示对用户可理解。
- 需要避免再次破坏 `Esc` 取消。

### 高精度语音包状态

已有能力：
- 已有下载、删除、预热和状态展示。
- `LocalSpeechModelReadiness` 已尝试合并磁盘状态和运行时状态。
- 状态文案已统一使用“高精度语音包”。

当前缺口：
- 需要增加版本或清单概念，支持更新提示。
- 状态必须来自真实状态，不能出现“准备好”和“尚未准备好”同时出现。
- 网络不可用或更新检查失败时，要显示“暂时无法检查”，不能误报失败。

## 1.1.0 产品需求

### P0：常用词库产品化收口

用户可以在设置里维护常用词。常用词应覆盖人名、项目、产品、技术词、公司/组织和其他。

验收：
- 添加、删除、导入常用词都能在设置页明确反馈。
- 重启后常用词仍存在。
- 常用词能影响后续识别上下文和术语修正。
- 文案不出现 hotwords、训练、模型学习等技术词。

### P0：确认式学习建议收口

ReadyType 可以建议用户把词加入常用词，但必须由用户确认。

验收：
- 建议区清楚说明“这会帮助以后识别更准”。
- 用户可以确认或忽略。
- 忽略后不会短时间重复提示同一建议。
- 不保存完整句子、私人聊天或邮件正文。

### P0：App 场景语气

输出必须符合场景。

验收：
- 微信/聊天：自然、简洁，不无根据添加“谢谢”“麻烦你了”等礼貌结尾。
- 邮件：当用户表达写邮件意图时，输出称呼、正文分段和合适结尾。
- 笔记：更适合条理化记录。
- AI 工具：输出清晰任务说明，不像普通聊天。

### P1：快捷键自定义体验复核

用户可以更换触发方式，但默认保持双击 `Option`。

验收：
- 修改后立即生效。
- 旧触发方式不再触发。
- `Esc` 取消不受影响。
- 文案清楚说明“开始说话”和“完成输入”，避免“录音”造成困惑。

### P1：高精度语音包更新提示

用户可以知道高精度语音包是否已安装、是否准备好、是否需要更新。

验收：
- 未安装、下载中、准备中、已准备好、检查失败、需要更新状态可区分。
- 已准备好后不会无故回退。
- 可以删除并重新下载。
- 更新检查失败时显示可理解的临时状态。

## 非目标

- 不新增 OpenAI、Anthropic、Gemini 等模型供应商。
- 不新增云端语音识别 API 配置。
- 不做静默学习。
- 不保存完整转写历史。
- 不把 Liquid Glass HUD 作为 1.1.0 阻塞项。

## 验证要求

- 先补单元测试，再改代码。
- 每个模块至少跑对应测试过滤器。
- 阶段性完成后跑 `swift test`。
- 打包前跑 `scripts/build-app.sh` 和 `scripts/package-dmg.sh`。
- 真实语音复测要记录：场景、原始识别、最终输出、耗时、粘贴结果。
