# ReadyType 1.4.0 需求说明：匿名产品统计

## 背景

ReadyType 现有使用概览只把每日完成次数、录音时长和输出字符数保存在用户本机。开发者无法判断首次使用漏斗、留存、功能采用率、失败率和等待时间，也无法用真实数据评估英文版。

## 产品目标

- 在不收集用户内容的前提下了解安装、首次成功输入、活跃和留存。
- 了解识别路线、输出方式、场景类别和高精度语音包的采用情况。
- 记录固定错误码和分桶耗时，用于发现回归。
- 向用户清楚说明收集范围，并提供随时关闭的开关。
- 保持现有本机 Dashboard，不把本机历史上传到 GitHub 或统计平台。

## 隐私要求

禁止上传音频、原始转写、最终输出、窗口标题、文件路径、剪贴板、常用词、DeepSeek 请求与响应、API Key、联系人或账号信息。App 只允许发送 `ANALYTICS_SPEC.zh-CN.md` 中列出的事件、枚举属性和经审计的 TelemetryDeck SDK 固有兼容性元数据。

## 验收标准

- 未配置统计服务时，网络发送为零，核心输入流程不受影响。
- 用户关闭匿名统计后，新事件立即停止发送。
- 事件属性不能接受自由文本。
- 错误只发送固定错误码，不发送 `localizedDescription`。
- 统计失败不能阻塞启动、录音、识别、整理或粘贴。
- 单元测试覆盖设置持久化、事件白名单、关闭状态和 No-op 行为。

## 非目标

- 不做录屏、Session Replay、用户画像、广告归因或跨站追踪。
- 不上传现有 `UsageStatistics.json`。
- 不在客户端保存管理 Token 或 Dashboard 凭据。
- 不在本版本实现英文 UI、英文 ASR 或中英混说。
