# ReadyType 1.3.0 需求说明：热门词包

## 背景

用户希望 ReadyType 像成熟输入法一样，自动跟上近期常用词，例如新上映电影名、热门科技产品、赛事名和网络流行词，同时不打扰使用、不牺牲速度、不污染用户自己的常用词。

公开可验证的成熟输入法方案显示，稳定输入体验通常来自分层词库：本地基础词库、用户词库、扩展词库和云候选。云端或热门词通常作为补充候选，而不是直接覆盖本地结果。

## 产品目标

- 用户不需要手动导入近期热门词。
- 语音输入时不出现额外等待、弹窗或配置负担。
- 近期专有名词识别更准，尤其是影视娱乐、科技产品和体育赛事。
- 热门词不会覆盖用户常用词，也不会把普通表达误改成热门词。
- 用户可以关闭自动更新、删除已下载词包、查看最后更新时间。

## 用户文案

使用“热门词包”，不使用“云词库”“训练”“模型同步”“热词注入”等技术化词。

建议设置文案：

- 标题：热门词包
- 说明：ReadyType 会在后台更新公开来源整理的近期常用词，用于提升识别和整理效果；不会上传你的输入内容。
- 状态：已更新 / 正在更新 / 暂时无法更新 / 已关闭
- 操作：自动更新热门词、立即更新、删除词包

## 功能范围

### 1. 分层词库

优先级从高到低：

1. 用户手动添加的常用词。
2. 用户确认过的常用词建议。
3. ReadyType 内置产品词、技术词和工作短语。
4. 当前 App / 场景相关词。
5. 热门词包。

热门词包永远不能高于用户词。

### 2. 后台更新

- 默认在空闲时自动更新。
- 更新失败不提示弹窗，不影响当前语音输入。
- 设置页保留状态和手动更新入口。
- 更新频率默认每日最多一次。
- 使用 ETag、版本号或内容 hash 避免重复下载。

### 3. 本地缓存

热门词包保存在用户目录：

`~/Library/Application Support/ReadyType/HotVocabulary/`

不写入仓库，不写入日志，不上传用户输入。

### 4. 词条元数据

每个词条至少包含：

- `value`：显示词。
- `aliases`：常见别名或中英混说形式。
- `category`：例如影视娱乐、科技产品、体育赛事、网络流行词。
- `scopes`：适用场景，例如聊天、文档、AI 工具。
- `source`：公开来源标识。
- `weight`：词条权重。
- `expiresAt`：过期时间。

### 5. 动态候选选择

每次语音输入前只选择少量候选词：

- Apple Speech contextual strings：最多 20-60 个热门词参与，且总候选不超过现有 100 个上限。
- AI 整理和翻译：只把高相关词作为术语候选提示。
- 直接转文字：只做保守后处理，不进行激进替换。

### 6. 隐私和安全

- 不上传用户转写、最终输出、App 名称、窗口标题或个人词库。
- 客户端不内置第三方私有 API key。
- 词包来源应由 ReadyType 维护侧生成，客户端只下载整理后的 manifest。
- 词包需要 schema 版本和内容 hash；后续可增加签名。

## 非目标

- 不做全网实时热榜抓取。
- 不抓取未授权网页或私有热搜接口。
- 不把热门词直接加入用户常用词。
- 不根据用户输入自动上报或训练热门词。
- 不在当前阶段做复杂词包市场。
- 不承诺本地 WhisperKit 对热门词实时偏置；第一阶段优先支持 Apple Speech contextual strings 和后处理。

## 验收标准

- 关闭网络时，语音输入不受影响。
- 热门词包更新失败时，只在设置页显示“暂时无法更新”。
- 用户词和确认式学习词优先级高于热门词。
- 热门词过期后不会继续参与候选。
- 每次识别候选选择耗时保持在现有上下文词汇性能预算内。
- 微信/聊天场景不会因为热门词包把普通表达误改成电影名或产品名。
