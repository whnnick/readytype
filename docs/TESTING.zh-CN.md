# 测试说明

这份说明适合发给第一次试用 ReadyType 的用户。

如果你需要直接转发给测试用户，可以使用 [测试邀请文案](./TESTER_INVITE.zh-CN.md)。

## 安装

1. 从 Releases 下载 `ReadyType.dmg`。
2. 打开 DMG，把 `ReadyType.app` 拖到 Applications。
3. 首次打开时，如果 macOS 提示无法验证开发者，请右键 `ReadyType.app`，选择“打开”，再确认“打开”。

不要关闭 macOS 系统安全设置。

## 首次配置

1. 打开 ReadyType。
2. 在设置里填写自己的 DeepSeek API Key。
3. 按提示授权麦克风、语音识别和辅助功能。
4. 如果需要更准确的长文本和术语识别，可以在设置里下载高精度语音包。

语音识别本身不需要额外填写云端语音 API 密钥。

## 怎么使用

1. 把光标放到任意输入框。
2. 双击 `Option`，开始说话。
3. 再次双击 `Option`，完成并输出。
4. 如果不想继续，按 `Esc` 取消。

如果没有自动粘贴，ReadyType 会把结果复制到剪贴板，可以手动 `Command + V`。

## 建议测试的内容

- 在微信、备忘录、浏览器、邮件或文档工具里输入。
- 分别测试直接转文字、整理成文、翻译成英文和写给 AI。
- 测试短句、长句、中英混说、人名、项目名和专业词。
- 测试 `Esc` 取消和自动粘贴。
- 测试高精度语音包准备好前后的识别体验。

## 反馈方式

请在 GitHub Issues 反馈：

https://github.com/whnnick/readytype/issues

建议说明：

- ReadyType 版本
- macOS 版本
- 使用的 App
- 输出方式
- 你说了什么
- ReadyType 输出了什么
- 你期望的结果

提交前请删除 API Key、私人聊天、私人邮件、真实客户信息和敏感业务内容。

## 维护者真实语音验收

- 开始前固定用例数量和停止条件，不在测试中无限追加相似句子。
- 每条只记录：场景、口述文本、原始识别、最终输出、耗时、粘贴结果。
- 失败先归类为识别、常用词/术语、AI 整理、语言格式或粘贴，再修改对应层。
- 修复后复测失败用例、一个相邻正例和一个防误改反例；发布前仍需执行完整门禁。
